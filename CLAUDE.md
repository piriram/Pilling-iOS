# Claude Code 프로젝트 설정

## 작업 방식

- 긴 작업을 진행할 때 중간에 확인을 요청하지 말고 끝까지 완료할 것
- 여러 파일을 수정하는 작업도 한 번에 진행할 것

## Git 커밋 규칙

- 커밋 메시지에 이모지를 사용하지 말것
- 커밋 메시지는 한글로 작성
- 간결하고 명확하게 작성
- **본문 없이 제목만 작성**
- **Co-Authored-By, Generated with Claude Code 등 공동 저자 정보 추가 금지**

### 커밋 메시지 포맷
```
Feat: 제목
```

타입 예시: Feat, Fix, Refactor, Chore, Docs 등

예시:
```
Feat: PillStatus 구조 개선
```

## 문서 작성 규칙

- **작업 완료 후 문서화하지 말 것**
- **별도로 요청하지 않는 한 README, 문서, 가이드 등을 생성하지 말 것**
- 마크다운 파일은 간결하게 작성
- 코드 주석과 간단한 설명 위주로 작성

## 코드 스타일

- Swift 코드는 기존 프로젝트 스타일 따르기
- 불필요한 주석 추가하지 말 것
- 이모지는 코드나 커밋에 사용하지 말 것
