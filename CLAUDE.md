# Claude Code 프로젝트 설정

## 작업 방식

- 긴 작업을 진행할 때 중간에 확인을 요청하지 말고 끝까지 완료할 것
- 여러 파일을 수정하는 작업도 한 번에 진행할 것

## Git 커밋 규칙

- 커밋 메시지에 이모지를 사용하지 말것
- 커밋 메시지는 한글로 작성
- 간결하고 명확하게 작성

### 커밋 메시지 포맷
```
[타입] 제목

내용 (필요시)
```

예시:
```
Refactor: PillStatus 구조 개선

TimeContext를 도입하여 상태 관리 단순화
```

## 문서 작성 규칙

- 마크다운 파일은 간결하게 작성
- README나 문서는 꼭 필요한 경우가 아니면 생성하지 말 것
- 코드 주석과 간단한 설명 위주로 작성

## 코드 스타일

- Swift 코드는 기존 프로젝트 스타일 따르기
- 불필요한 주석 추가하지 말 것
- 이모지는 코드나 커밋에 사용하지 말 것
